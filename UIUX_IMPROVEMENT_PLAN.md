# Together Pray UI/UX 개선 계획

## 분석 결과 요약

### 현재 상태
- TailwindCSS 기반 커스텀 스타일링 (UI 라이브러리 없음)
- 기본적인 반응형 디자인 구현됨
- 심플한 그레이스케일 + 블루 액센트 컬러
- 로딩 스켈레톤 없음 (텍스트만 표시)
- 다크모드 미지원
- 마이크로 인터랙션 부재

### 경쟁 앱 분석 (Echo Prayer, PrayerMate, Glorify)
| 앱 | 강점 |
|---|------|
| **Echo Prayer** | 심플한 기도 추적, 타이머, 응답된 기도 관리, 그룹 공유 |
| **PrayerMate** | 기도 리스트 정리, 스케줄 리마인더, 이미지/PDF 첨부 |
| **Glorify** | 성경적 명상, 저널링, 게이미피케이션, 매력적인 UI |

### 2025 UI/UX 트렌드
1. **Glassmorphism** - 반투명 유리 효과 (Apple Liquid Glass)
2. **다크모드** - 기본 기대치 (필수)
3. **마이크로 애니메이션** - 부드러운 전환 효과
4. **Thumb-friendly** - 하단 네비게이션, 엄지 영역 최적화
5. **접근성** - 대비, 텍스트 크기, 음성 지원

## 개선 계획

### Phase 1: 디자인 시스템 & 테마 (기초)

#### 1.1 다크모드 지원
- [ ] CSS 변수 기반 테마 시스템 구축
- [ ] `prefers-color-scheme` 미디어 쿼리 적용
- [ ] 토글 버튼 추가 (시스템 / 라이트 / 다크)
- [ ] 로컬스토리지에 테마 설정 저장

#### 1.2 컬러 시스템 개선
```css
/* 라이트 모드 */
--bg-primary: #FFFFFF;
--bg-secondary: #F9FAFB;
--text-primary: #111827;
--text-secondary: #6B7280;
--accent-blue: #3B82F6;
--accent-green: #10B981;
--accent-yellow: #F59E0B;

/* 다크 모드 */
--bg-primary: #1F2937;
--bg-secondary: #111827;
--text-primary: #F9FAFB;
--text-secondary: #9CA3AF;
```

#### 1.3 타이포그래피 개선
- [ ] Pretendard 또는 Noto Sans KR 웹폰트 적용
- [ ] 폰트 스케일 시스템 정의 (text-xs ~ text-4xl)

---

### Phase 2: 네비게이션 & 레이아웃

#### 2.1 하단 네비게이션 바 추가 (모바일)
```
┌─────────────────────────────────────┐
│                                     │
│         (콘텐츠 영역)               │
│                                     │
├─────────────────────────────────────┤
│  🏠 홈  │  🙏 기도  │  ✅ 응답  │  👤 내정보 │
└─────────────────────────────────────┘
```
- [ ] 하단 고정 네비게이션 컴포넌트 생성
- [ ] 현재 페이지 하이라이트
- [ ] 부드러운 아이콘 애니메이션

#### 2.2 상단 헤더 개선
- [ ] 글래스모피즘 효과 적용 (backdrop-blur)
- [ ] 스크롤 시 축소되는 헤더
- [ ] 그룹 이름 + 검색 아이콘

---

### Phase 3: 컴포넌트 개선

#### 3.1 기도 카드 리디자인
현재:
```
┌────────────────────────────┐
│ Title              [Badge] │
│ Content...                 │
│ Author | Category | Count  │
│ [Pray Button]              │
└────────────────────────────┘
```

개선:
```
┌────────────────────────────┐
│ 👤 Author        2시간 전  │
│ ──────────────────────────│
│ Title                      │
│ Content (3줄)...           │
│                            │
│ [상태 Badge] [카테고리]    │
├────────────────────────────┤
│ 🙏 15명 함께 기도  [기도하기]│
└────────────────────────────┘
```

- [ ] 작성자 정보 상단 배치
- [ ] 기도 수 시각적 강조
- [ ] 카드 터치 피드백 (ripple effect)
- [ ] 스와이프 액션 (빠른 기도하기)

#### 3.2 기도 버튼 개선
- [ ] 펄스 애니메이션 (기도 유도)
- [ ] 기도 완료 시 축하 애니메이션 (confetti 또는 하트)
- [ ] 햅틱 피드백 지원 준비

#### 3.3 로딩 스켈레톤
- [ ] 기도 카드 스켈레톤 컴포넌트
- [ ] 페이지별 로딩 UI 적용
- [ ] 부드러운 shimmer 애니메이션

#### 3.4 빈 상태 (Empty State) 개선
- [ ] 일러스트레이션 또는 아이콘 추가
- [ ] 동기부여 문구
- [ ] CTA 버튼 강조

---

### Phase 4: 인터랙션 & 애니메이션

#### 4.1 페이지 전환 애니메이션
- [ ] 페이지 진입/퇴장 fade + slide 효과
- [ ] Next.js App Router 레이아웃 활용

#### 4.2 마이크로 인터랙션
- [ ] 버튼 호버/클릭 피드백
- [ ] 상태 변경 시 부드러운 전환
- [ ] 스크롤 기반 요소 등장 애니메이션

#### 4.3 Pull-to-Refresh (모바일)
- [ ] 아래로 당겨서 새로고침 기능
- [ ] 커스텀 로딩 인디케이터

---

### Phase 5: 기능 UX 개선

#### 5.1 기도 작성 플로우 개선
- [ ] 단계별 폼 (선택사항)
- [ ] 카테고리 비주얼 선택 (아이콘 + 텍스트)
- [ ] 자동 저장 (드래프트)
- [ ] 작성 완료 시 격려 메시지

#### 5.2 기도 상태 업데이트 UX
- [ ] 상태 변경 시 축하 모달 (응답 완료)
- [ ] 타임라인 시각화 개선
- [ ] 업데이트 알림 badge

#### 5.3 검색 & 필터 개선
- [ ] 검색 바 상단 고정
- [ ] 필터 칩 스크롤
- [ ] 정렬 옵션 (최신/기도많은순/오래된순)

- [ ] 포커스 인디케이터 개선

#### 6.2 성능 최적화
- [ ] 이미지 lazy loading
- [ ] 컴포넌트 코드 스플리팅
- [ ] 가상 스크롤 (긴 목록)

## 기술 스택 추가 제안

### 필수
- `framer-motion`: 애니메이션 라이브러리
- `lucide-react`: 아이콘 라이브러리
- `@fontsource/pretendard`: 한글 웹폰트

### 선택사항
- `react-spring`: 물리 기반 애니메이션
- `canvas-confetti`: 축하 효과

## 작업 순서

1. **기초 설정** (Phase 1)
   - 다크모드 테마 시스템
   - 웹폰트 설정
   - framer-motion, lucide-react 설치

2. **레이아웃 개선** (Phase 2)
   - 하단 네비게이션
   - 헤더 개선

3. **핵심 컴포넌트** (Phase 3)
   - 로딩 스켈레톤
   - 기도 카드 리디자인
   - 기도 버튼 애니메이션

4. **인터랙션** (Phase 4)
   - 마이크로 애니메이션
   - 페이지 전환

5. **마무리** (Phase 5-6)
   - 접근성 검토
   - 성능 최적화
